A feature comparison table would be accurate here. It would also miss the point.

The more important distinction is not individual features. It is the difference between a system that was digitised and a system that was designed. Most of what follows is not a new capability. It is the correct implementation of something that conceptually existed — but was never built into the infrastructure where students actually need it.

## The Distinction That Changes Everything

Every row in that table is a real improvement. But one of them changes the nature of the system rather than improving a part of it: one-time registration, valid across all participating counsellings.

In the current system, every counselling is a separate entry point. A student navigating three or four counsellings simultaneously is, from the perspective of each one, a new and unverified person. They re-register. They re-upload. They are re-verified. The system has no memory of them across contexts.

This is not a usability problem. It is an architectural one. And it produces every downstream consequence: the 187 hours, the 4.2 document submissions, the Rs. 21,900 in pre-tuition costs, the 56% portal crash rate during peak windows. All of it flows from the absence of a shared identity layer.

Superadmission introduces that layer. A student creates one verified profile. Every participating counselling reads from it. The student is never asked to start over.

---

## What This Does Not Change

Superadmission does not change eligibility criteria, reservation policy, or counselling authority. States and bodies retain full control over their own processes, schedules, and rules.

It is worth being direct about this because centralisation and unification are frequently conflated. They are not the same thing. Superadmission is not a proposal to centralise admissions. It is a proposal to make the infrastructure beneath decentralised admissions work properly. The authority stays where it is. The execution becomes shared.

## The Allocation Engine

The allocation logic in the current system varies by counselling, with different levels of transparency and almost no explainability for students. You receive an outcome. You are not told why.

Superadmission runs allocation on stable matching — a well-studied algorithm used in medical residency systems globally, including the US National Resident Matching Program which has run continuously since 1952. It has one property that matters above all others: it is strategy-proof. Expressing honest preferences always produces the best possible outcome for a student. There is no advantage to gaming the system.

> The same algorithm that matches medical residents to hospitals in the US has been processing matches since before the internet existed. It works. The math has been verified for decades.

Every allocation comes with a full decision trail — inputs, constraints, sequence. A student who did not receive their first preference can see exactly why. Not as a response to a grievance. As a feature, available to every student, automatically.

This changes the grievance dynamic at the source. Most admission grievances arise from unexplained outcomes. When the outcome is explained, the grievance often does not need to be filed.
