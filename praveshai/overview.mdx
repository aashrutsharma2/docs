PraveshAI™ is not a chatbot.

It is the engine powering every workflow in Superadmission — for students navigating admissions, for reviewers handling document queues, and for counselling authorities running allocation. Every decision it touches is logged, explainable, and auditable.

## What PraveshAI™ handles

<CardGroup cols={3}>
  <Card title="Identity" icon="fingerprint">
    One-time Aadhaar-linked verification. Profile created once, read by every participating counselling.
  </Card>
  <Card title="Documents" icon="file-shield">
    Verified at source. Confidence-scored before human review. Reused across counsellings.
  </Card>
  <Card title="Guidance" icon="compass">
    Eligibility checks, smart deadline alerts, choice-filling probability — per student.
  </Card>
  <Card title="Allocation" icon="scale-balanced">
    Merit-cum-preference matching. Configurable per authority. Every outcome traceable.
  </Card>
  <Card title="Coordination" icon="arrows-split-up-and-left">
    State management across multiple counselling workflows. Deadline sync.
  </Card>
  <Card title="Audit" icon="magnifying-glass-chart">
    Every decision logged. Every action hash-stamped. Student-readable. Authority-verifiable.
  </Card>
</CardGroup>

## System layers

| Layer | What it does |
|---|---|
| Identity and Verification | Aadhaar-aligned profile creation, document source verification |
| Document Workflows | Fetch, verify, confidence-score, reuse across counsellings |
| Guidance Engine | Eligibility checks, smart alerts, choice-filling probability |
| Seat Allocation | Merit-based matching, round logic, per-authority rule configuration |
| Workflow Coordination | Cross-counselling state management, deadline sync |
| Audit and Explainability | Decision logging, student-readable and authority-verifiable records |
