Students navigating India's admissions cycle participate in multiple independent systems — JoSAA, State CETs, institutional counsellings — with **no coordination between any of them**. Separate accounts. The same documents uploaded three, four, five times. Deadlines that collide. Seat decisions made in one system with no visibility into the others.

Superadmission is a **proposed shared infrastructure layer** above these systems. It unifies identity, documents, deadlines, and guidance — **without touching how any authority allocates seats**.

## The Proposed Shift

JoSAA allocates IIT and NIT seats. MCC allocates AIQ medical seats. State CETs allocate within their jurisdiction. **None of this changes.**

What changes is what happens before and between those systems.

| Today | Proposed |
| --- | --- |
| New account created per counselling | One Aadhaar-linked profile, built once |
| Same documents uploaded 4.1 times per cycle on average | Fetched from DigiLocker once, verified once, accepted everywhere |
| Deadlines manually tracked across portals | Unified dashboard with proactive alerts at 72h, 24h, and 6h |
| Allotment status checked per portal independently | Single view, side-by-side comparison across all active counsellings |
| Guidance from paid consultants — Rs 5,000 to Rs 2 lakh | PraveshAI at every decision point, available to every student |

The figures above come from approximately **2,000 student journeys** tracked through CollegeCult operations.

## Built on India's Existing Infrastructure

Superadmission does not build parallel infrastructure where public infrastructure already exists. Each proposed layer maps to a component of India Stack.

<CardGroup cols={4}>
  <Card title="Aadhaar" icon="fingerprint" href="/blueprint/public-infrastructure-alignment">
    Optional e-KYC and OTP-based authentication. Identity entered once at profile creation.
  </Card>

  <Card title="DigiLocker" icon="file" href="/blueprint/public-infrastructure-alignment">
    Mark sheets and rank cards fetched from issuing authorities with explicit student consent.
  </Card>

  <Card title="UPI" icon="indian-rupee" href="/blueprint/public-infrastructure-alignment">
    All registration and acceptance fees across every active counselling — one interface.
  </Card>

  <Card title="APAAR" icon="id-card" href="/blueprint/public-infrastructure-alignment">
    Persistent student record aligned with the Academic Bank of Credits under NEP 2020.
  </Card>
</CardGroup>

The structural approach follows the same pattern as UPI for payments and ONDC for commerce — **a shared protocol layer above independent existing actors, with no actor replaced**.

## Scale

**23 lakh students** appeared for JEE Main in 2024. **24 lakh** for NEET UG. Each participating in an average of 2.3 separate counselling registrations per cycle, submitting the same documents an average of 4.1 times.

NEP 2020 targets 50% Gross Enrolment Ratio by 2035 against a current rate of 28.4%. **Volume into this process grows every year. The coordination infrastructure has not.**
